# Artifacts

This directory contains vulnerability reports (example artifacts) generated by our system. These files are stored in structured Markdown reports for easy review, archiving, and presentation.

- **T1 (Find Hard-Coded Credentials):** This task scans the entire firmware for passwords, keys, and secrets embedded in the code.
- **T2 (Identify Third-Party Components and CVEs):** This task identifies libraries, binaries, and executables in the firmware and maps them to known vulnerabilities in a CVE database.
- **T3 (Trace NVRAM/Environment Variable Interactions):** This task traces the use of environment variables in the firmware and identifies unsanitized flows leading to dangerous calls.
- **T4 (Perform Web Attack Chain Analysis):** This task analyzes the firmware's web server, evaluates its attack surface, and traces user input to risky functions.
- **T5 (Detect and Verify Vulnerabilities):** This task performs a comprehensive analysis of the firmware filesystem, identifies dangerous functions, traces function call paths, and verifies taint-style vulnerabilities.

## üìã Report Content Description

Each `.md` report typically includes:

1. Binary and firmware information: Vendor, model/firmware version, binary path.
2. Vulnerability type (if inferred): CWE classification or free-text description.
3. Discovery summary and evidence: Includes taint propagation chains, key code locations, or r2 output snippets.
4. Reproduction/verification instructions (if available) and reference links.
5. Severity assessment recommendations.

These reports are generated by automated analysis scripts, and some classifications (e.g., CWE) may be heuristic‚Äîmanual review is required.
